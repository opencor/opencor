Known Issues/Limitations

Please note that the following known issues/limitations have been observed on Windows 7, Ubuntu 11.04 (Natty Narwhal; both 32-bit and 64-bit) and Mac OS X 10.6 (Snow Leopard). So, if you use a different operating system, these issues/limitations may or not be relevant to you.

Windows, Linux and Mac OS X

By default, OpenCOR uses your system's language for menus, dialog boxes, etc., as long as it is either English or French (please contact us if you would like another language to be added). If your system uses another language, then OpenCOR will default to English. Otherwise, if you specifically specify English or French as the language to be used by OpenCOR, then please be aware that system messages, dialog boxes, etc. will be displayed in your system's language.

Windows

  • OpenCOR's embedded file browser may, on some computers, result in OpenCOR being slow to respond at startup. This is, unfortunately, a very well-known problem with Windows and for which there are several known causes and solutions.
  • Resetting all the user settings (by selecting the Tools|Reset All menu) requires restarting OpenCOR. There may, however, be times when OpenCOR will have to restarted manually. We believe this to be a problem with Windows (indeed, this problem has not been seen on neither Linux nor Mac OS X), even though we have yet to find its cause.

Linux

Depending on your window manager (e.g. Compiz), OpenCOR may or not experience one of the following issues/limitations (none of which affect the good functioning of the software itself):

  • Upon starting OpenCOR, the software will try to retrieve your previous session's settings which may result in the main window showing up lower down the screen than it should.
  • To show the help window by pressing the F1 key may, if the help window is to be floating (as opposed to being docked to the main window), result in OpenCOR not being the active application anymore. This means that if you were to press the F1 key one more time, the help window will remain visible while it should have disappeared.
  • Menu items that have sub-menus may not, on some versions of Gnome, have their icon displayed (should they normally have one). This has been discussed in many places, such as here and here. Anyway, one way to change this behaviour is to enter the following on the command line: 
    $ gconftool-2 --type Boolean --set /desktop/gnome/interface/menus_have_icons True

Note that the above issues have nothing to do with OpenCOR itself (everything works fine on both Windows and Mac OS X), but with the window manager that you use. Indeed, there is a lack of compliancy between the various window managers, so something that works with one window manager may or not work with another, as is unfortunately the case here.
