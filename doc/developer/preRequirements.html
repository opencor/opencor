Pre-requirements

The following tools are required in order to develop, build, test and/or package OpenCOR:

  • Git (the git package on Ubuntu): a distributed version control system. You can get a copy of OpenCOR's source code by cloning our Git repository either in read-only mode: 
    $ git clone git://github.com/opencor/opencor.git
    or in read-write mode: 
    $ git clone git@github.com:opencor/opencor.git
    Note #1: you might want to check the GitHub help on setting up Git on Windows, Linux and Mac OS X, respectively.
    Note #2: Git can be used either from the command line or with TortoiseGit, QGit and GitX on Windows, Linux and Mac OS X, respectively.
  • CMake 2.8+ (the cmake package on Ubuntu): a cross-platform build system;
  • C++ toolchain: a set of C++ tools. The toolchain is different depending on the platform you are using:
    • Windows: MinGW 4.4.0 is required, but it comes with the Qt SDK (see below), so no need to do anything here;
    • Linux: the g++ and libglu1-mesa-dev packages on Ubuntu; and
    • Mac OS X: Xcode 4.1.
  • Qt SDK 1.1.4: this combines Qt 4.8.0 (a cross-platform application and UI framework) and Qt Creator 2.4.0 (a cross-platform environment for Qt), among other things:
    • Windows;
    • Linux 32-bit/64-bit; and
      Note: we would recommend sudo installing the SDK, so that it gets installed under under /opt/QtSDK rather than under your home directory.
    • Mac OS X: you might want to install the SDK under /Developer rather than your home folder as suggested by the installer.
    Note: the Qt SDK will, by default, install everything that is needed to build OpenCOR (except, on Windows, for the MinGW 4.4 package which needs to be manually selected from the Qt SDK|Miscellaneous category), so it is pretty much up to you which package(s) you want to install (you can always add/remove packages later on, if wanted/needed). Just for reference, here are the packages that we selected for our Windows build (it is very similar on Linux and Mac OS X).
  • NSIS (Windows only): a program for creating Windows installers.

Note that our build environment requires the version of Qt mentioned above, i.e. 4.8.0. It may be that OpenCOR could be built using an earlier version of Qt, but this is not supported (and not recommended either). Still, should you want to give it a try, then look for FIND_PACKAGE(Qt4 4.8.0 REQUIRED) in [OpenCOR]/cmake/common.cmake and replace 4.8.0 with whatever version of Qt that you want to use.

Finally, you might want/need to update your PATH environment variable by adding the following paths:

  • Windows: C:\Program Files (x86)\CMake 2.8\bin (or C:\Program Files\CMake 2.8\bin if you are running a 32-bit version of Windows), C:\QtSDK\mingw\bin and C:\QtSDK\Desktop\Qt\4.8.0\mingw\bin;
  • Linux: /opt/QtSDK/Desktop/Qt/4.8.0/gcc/bin (in /etc/environment on Ubuntu); and
  • Mac OS X: /Developer/QtSDK/Desktop/Qt/4.8.0/gcc/bin (in /etc/profile).
